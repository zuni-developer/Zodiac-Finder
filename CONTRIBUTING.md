# Contributing Guidelines

I appreciate your interest in contributing. 
This project is a simple, visually polished web app built with **HTML, CSS, and Vanilla JavaScript**, focusing on clean UI, smooth UX, and clarity.

Contributions that enhance visuals, usability, or learning value are always welcome.

---

## Project Focus

This project prioritizes:
- Clean and aesthetic UI
- Smooth user experience
- Beginner-friendly JavaScript
- Minimal logic (no frameworks)
- Readable and well-organized code

Please keep changes aligned with this direction.

---

## Ways You Can Contribute

You may contribute by:

- Improving UI design or styling
- Adding subtle animations or transitions
- Improving zodiac calculation logic
- Adding zodiac descriptions or traits
- Fixing bugs or edge cases
- Improving accessibility (contrast, labels, focus states)
- Adding light/dark mode support

---

## Project Structure

Please respect the existing folder structure:

```

Zodiac-Finder/
├── index.html
├── src/
├── css/
├── img/
└── README.md

````

If you add new files:
- Keep them logically organized
- Update the README if necessary

---

## Coding Guidelines

- Keep logic easy to follow
- Use meaningful variable names
- Add comments where logic may confuse beginners

---

## UI & UX Guidelines

- Maintain the cosmic / zodiac theme
- Ensure layouts look good on modern browsers
- Keep animations subtle and smooth
- Avoid cluttered or distracting UI elements

---

## Testing

Before submitting your changes:
- Test on both pages (index & result)
- Verify zodiac sign calculation for multiple dates
- Check layout responsiveness

---

## How to Contribute

1. **Fork** the repository
2. **Clone** your fork:
  ```bash
   git clone https://github.com/zuni-developer/Zodiac-Finder.git
  ```

3. Create a new branch:

  ```bash
   git checkout -b feature/your-feature-name
  ```
4. Make your changes
5. Commit with a clear message:

  ```bash
   git commit -m "Add zodiac personality descriptions"
  ```
6. Push your branch:

  ```bash
   git push origin feature/your-feature-name
  ```
7. Open a **Pull Request** with:

   * Description of your changes
   * Screenshots (for UI changes)
